1. SO 注入的基本原理

SO 注入是一种在 Android/Linux 环境下通过将自定义动态链接库加载到目标进程地址空间中来改变其行为的技术。常用于游戏外挂、逆向分析、安全测试等场景。

基本原理是利用操作系统提供的动态链接机制（如 dlopen），在目标进程中执行任意代码。注入流程通常分为两步：

  1. 在目标进程中调用 dlopen 加载恶意 SO 文件
  2. SO 的 __attribute__((constructor)) 函数自动执行，完成 hook 或篡改

2. 常见注入方法

方法一：ptrace + dlopen 注入

利用 ptrace 系统调用 attach 到目标进程，修改其寄存器和栈，使其执行 dlopen：

// 1. attach 目标进程
ptrace(PTRACE_ATTACH, target_pid, NULL, NULL);
waitpid(target_pid, NULL, 0);

// 2. 保存目标进程的寄存器状态
struct user_regs_struct old_regs, new_regs;
ptrace(PTRACE_GETREGS, target_pid, NULL, &old_regs);


// 3. 在目标进程中分配内存（用于存放 SO 路径字符串）
//    通过 mmap 系统调用在远程进程分配空间
new_regs = old_regs;
new_regs.orig_rax = 9;  // __NR_mmap
new_regs.rdi = 0;       // addr = NULL
new_regs.rsi = 4096;    // length
new_regs.rdx = PROT_READ | PROT_WRITE | PROT_EXEC;  // prot
new_regs.r10 = MAP_ANONYMOUS | MAP_PRIVATE;  // flags
new_regs.r8 = -1;       // fd
new_regs.r9 = 0;        // offset


// 4. 注入 shellcode 调用 dlopen/dlsym
// 5. 恢复寄存器，detach 进程
ptrace(PTRACE_SETREGS, target_pid, NULL, &old_regs);
ptrace(PTRACE_DETACH, target_pid, NULL, NULL);

方法二：LD_PRELOAD 预加载

设置环境变量 LD_PRELOAD 使动态链接器在程序启动时优先加载指定 SO：

LD_PRELOAD=./libhook.so ./target_app

该方法只能在进程启动前设置，适用于 hook libc 函数（如 open、read、connect）的场景。
