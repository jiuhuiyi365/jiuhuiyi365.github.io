解决 Kali Linux 无法 git clone 的问题

1. 问题现象

在 Kali Linux 中执行 git clone 时，可能遇到以下错误：

fatal: unable to access 'https://github.com/.../repo.git/':
GnuTLS recv error (-110): The TLS connection was non-properly terminated.

或


error: RPC failed; curl 28 OpenSSL SSL_read: Connection was reset


或


Cloning into ‘repo’…
fatal: unable to access ’…’: Failed to connect to github.com port 443:
Connection timed out

这些问题在 Kali Linux（国内网络环境）中非常常见，主要由以下原因导致：

  • DNS 解析被污染或超时
  • GitHub 的 HTTPS 端口（443）被墙
  • Git 底层使用的 GnuTLS 库与服务器不兼容
  • 系统代理配置缺失

2. DNS 配置优化

修改 /etc/resolv.conf 使用可靠的 DNS 服务器：

# 备份原始配置
sudo cp /etc/resolv.conf /etc/resolv.conf.bak

写入新的 DNS 配置


sudo tee /etc/resolv.conf << ‘EOF’
nameserver 8.8.8.8
nameserver 114.114.114.114
nameserver 223.5.5.5
EOF

注意
Kali Linux 使用 NetworkManager 管理网络，重启后 /etc/resolv.conf 可能被还原。如需永久修改，编辑 /etc/NetworkManager/NetworkManager.conf 并添加 dns=none。

3. 使用 SSH 替代 HTTPS

HTTPS 协议（443 端口）容易被干扰，而 SSH 协议（22 端口）通常可用：

# 1. 生成 SSH 密钥
ssh-keygen -t ed25519 -C "your_email@example.com"

2. 将公钥添加到 GitHub


cat ~/.ssh/id_ed25519.pub


复制内容 → GitHub Settings → SSH and GPG keys → New SSH key


3. 测试连接


ssh -T git@github.com


4. 使用 SSH 地址克隆


git clone git@github.com
/repo.git

若 22 端口也被封，可尝试使用 443 端口：

# 编辑 ~/.ssh/config
Host github.com
    Hostname ssh.github.com
    Port 443
    User git

4. 配置 Git 代理

如果本地有代理（如 Clash、V2Ray），可为 Git 单独配置：

# HTTP 代理
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

SOCKS5 代理


git config —global http.proxy socks5://127.0.0.1

git config —global https.proxy socks5://127.0.0.1


仅对 GitHub 使用代理


git config —global http.https://github.com.proxy http://127.0.0.1:7890


取消代理


git config —global —unset http.proxy
git config —global —unset https.proxy

5. GnuTLS 错误修复

如果遇到 GnuTLS recv error (-110)，可以将 Git 的 HTTP 后端从 GnuTLS 切换为 OpenSSL：

# 卸载系统自带的 git，从源码编译（使用 OpenSSL）
sudo apt remove git -y
sudo apt install -y build-essential libssl-dev libcurl4-openssl-dev \
    libexpat1-dev gettext libz-dev

cd /tmp
wget https://www.kernel.org/pub/software/scm/git/git-2.43.0.tar.gz
tar xzf git-2.43.0.tar.gz
cd git-2.43.0
make prefix=/usr/local all -j$(nproc) CURLDIR=/usr NO_GNUTLS=1
sudo make prefix=/usr/local install

6. 使用镜像加速

对于 GitHub 仓库，可以使用国内镜像加速克隆：

# GitHub 镜像（地址经常变动，需要搜索最新可用的）
git clone https://gitclone.com/github.com/user/repo.git
git clone https://ghproxy.com/https://github.com/user/repo.git

克隆后修改 remote 为原始地址


cd repo
git remote set-url origin https://github.com/user/repo.git
