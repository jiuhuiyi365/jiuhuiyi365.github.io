1. Transformer 架构

2017 年 Google 团队在论文 Attention Is All You Need 中提出了 Transformer 架构，彻底取代了 RNN/LSTM 在 NLP 领域的统治地位。核心组件包括：

  • Self-Attention（自注意力）：计算序列中每个位置与其他所有位置的相关性，实现全局信息捕获
  • Multi-Head Attention（多头注意力）：多个注意力头并行运算，捕获不同维度的依赖关系
  • FFN（前馈神经网络）：两层全连接网络，引入非线性变换
  • Residual Connection + LayerNorm：残差连接缓解梯度消失，层归一化稳定训练

注意力计算公式

Attention(Q, K, V) = softmax(Q·K^T / √d_k) · V

其中 d_k 为 Key 的维度，缩放因子防止 softmax 梯度消失


Q = X·W_Q,  K = X·W_K,  V = X·W_V

主流架构变体

架构代表模型特点
Encoder-OnlyBERT, RoBERTa双向注意力，适合分类/NER
Decoder-OnlyGPT, Claude, LLaMA因果注意力（Causal Mask），自回归生成
Encoder-DecoderT5, BART编码理解 + 解码生成，适合翻译/摘要
提示
理解 Transformer 并非使用大模型的必要条件，但有助于更好地设计 Prompt 和理解模型行为边界。

2. 自回归生成

当前主流 LLM（GPT-4, Claude, LLaMA 等）均采用自回归（Autoregressive）生成方式：逐 Token 预测下一个词的概率分布，然后采样一个 Token，将其拼接到输入后继续预测。

生成过程示例

输入: "今天天气"
Step 1: P("真"|今天天气) = 0.35, P("好"|今天天气) = 0.42, ...
        采样 → "好"
Step 2: P("。"|今天天气好) = 0.5, P("，"|今天天气好) = 0.3, ...
        采样 → "。"
输出: "今天天气好。"

采样策略

参数效果推荐场景
temperature=0贪心解码，每次选概率最高的词代码生成、精确任务
temperature=0.3~0.7较稳定，有一定随机性通用对话、问答
temperature=0.7~1.0高创造性，输出多样写作、头脑风暴
top_p=0.9Nucleus 采样，只从概率前 90% 中选控制输出质量
top_k=50只从概率前 50 个词中选限制离谱输出

3. Token 与分词

LLM 不直接处理字符，而是将文本切分为 Token（词片段）。主流分词方案：

  • BPE（Byte Pair Encoding）：GPT 系列使用，从字符级开始逐步合并高频对
  • SentencePiece：Google/Claude/LLaMA 使用，支持无监督分词，中日韩表现好
  • WordPiece：BERT 使用，类似 BPE 但使用似然而非频率

Token 数量估算

语言近似比例示例
英文~4 字符 = 1 Token"Hello world" ≈ 2 Token
中文~1-2 汉字 = 1 Token"你好世界" ≈ 3-4 Token
代码~3-5 字符 = 1 Token取决于语法复杂度
注意
中文 Token 消耗约为英文的 2-3 倍，直接影响 API 费用和上下文窗口的有效长度。
